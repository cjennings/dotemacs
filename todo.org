In Org-mode the ‚Äúagenda‚Äù machinery by default doesn‚Äôt pay any attention to your buffer¬≠narrow ‚Äî it always scans whole files in your =org-agenda-files=.  However, you can teach Org-agenda to only look between two points (the start and end of your current subtree) by using the built-in ‚Äúrestrict‚Äù hooks:

  1.  =org‚Äêagenda‚Äêrestrict=         ‚Äì turn on restriction
  2.  =org‚Äêagenda‚Äêrestrict‚Äêbegin=  ‚Äì a marker or position where scanning begins
  3.  =org‚Äêagenda‚Äêrestrict‚Äêend=    ‚Äì where it ends
  4.  (optionally) set =org-agenda-files= to just your current file

Here is a drop-in =org-agenda-custom-commands= entry that will give you a ‚ÄúSubtree TODOs‚Äù view limited to the subtree you‚Äôre on.  Put this in your Emacs init:

#+begin_src  emacs-lisp
;; -------------------------------------------------------------------
;; 1) Define a custom agenda command "C" (hit C-c a C)
;;    that shows only TODO entries in the current subtree.
;; -------------------------------------------------------------------
(with-eval-after-load 'org
  (add-to-list 'org-agenda-custom-commands
    '("C"                                ; the key you press after C-c a
      "Subtree TODOs"                   ; a descriptive name
      ((todo ""                         ; match any TODO keyword
         ;; === per-block settings ===
         ;; only look in this file
         (org-agenda-files (list buffer-file-name))
         ;; enable the restriction engine
         (org-agenda-restrict t)
         ;; start at the top of the current heading
         (org-agenda-restrict-begin
          (progn (org-back-to-heading t) (point)))
         ;; end at the end of this subtree
         (org-agenda-restrict-end
          (progn (org-end-of-subtree t) (point))))
       ;; you can add more blocks here if you like
       ))))
#+end_src

How this works, step by step:

  ‚Ä¢ We bind a new custom command under the key ‚ÄúC‚Äù (so you‚Äôll type =C-c a C=).  
  ‚Ä¢ In that command we use the =todo= matcher =""= to catch every headline whose state is one of your TODO keywords.  
  ‚Ä¢ We dynamically set  
     ‚Äì =org-agenda-files= to just the current buffer‚Äôs file,  
     ‚Äì =org-agenda-restrict= to non-nil so Org will honor the begin/end markers,  
     ‚Äì =org-agenda-restrict-begin= to the position of the current heading,  
     ‚Äì =org-agenda-restrict-end= to the end of the subtree.  

When you invoke it (=C-c a C=), Org will only scan headlines in that slice of the file and will build you a mini-agenda of TODO items from exactly that subtree.

Quick alternative: if you don‚Äôt need the full agenda machinery (dates, deadlines, etc.) but just want a fast outline of your TODOs under the current heading, you can also use the sparse-tree command:

  ‚Ä¢ =C-c / t=  ‚áí shows only the TODO headings in the narrowed or whole buffer as an indented outline.

‚Äî Tips & Pitfalls ‚Äî

  ‚Ä¢ Make sure your file is saved and in =org-agenda-files= (the snippet above forces it).  
  ‚Ä¢ Remember to call =org-narrow-to-subtree= (or let the command compute its own begin/end via =org-back-to-heading=/=org-end-of-subtree=).  
  ‚Ä¢ If you rename or move your file, Emacs must still see =buffer-file-name= valid.  
  ‚Ä¢ You can duplicate the above snippet for other TODO states, agenda views, or matchers (deadlines, tags, etc.)  

With this in place you effectively get an agenda tailored to exactly the bit of tree you‚Äôre working on.
